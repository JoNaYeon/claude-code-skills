# 책/독후감 인테이크

책은 기사와 동일한 4단계 검증을 거치되, 추가로 저자의 프레임워크를 추출하고 내 상황에 적용한다.

## 입력 방식

사용자가 책 내용을 제공하는 방식:
1. **직접 타이핑** — 읽은 내용을 자유롭게 기술
2. **markitdown으로 변환** — EPUB, PDF 등 전자책 파일을 마크다운으로 변환:
   ```bash
   markitdown book.epub > book_raw.md
   markitdown book.pdf > book_raw.md
   ```
3. **NotebookLM 요약 경유** — [2차요약] 태그 부여

## 4단계 비판적 검증

### ① 출처 검증
- 저자의 전문성, 학문적/실무적 배경
- 원서 vs 번역본 vs 요약본 구분
- `source_quality`: 원서/원저자 = primary, 요약본 = secondary

### ② 근거 검증
- 저자의 주장이 데이터/연구 기반인지, 경험론/의견인지
- 핵심 주장별 `[사실]` / `[해석]` / `[불명]` 태그

### ③ 반대 시나리오
- 저자의 프레임워크가 적용되지 않는 상황
- 다른 저자/학파의 반대 주장

### ④ 적용성
- **"이 프레임워크로 내 상황을 보면?"** — 핵심 질문
- 구체적인 행동 지침이나 의사결정에 연결

## 추가 처리: 프레임워크 추출

기사 인테이크와 차별화되는 부분:

### 저자의 핵심 프레임워크
- 전체 요약이 아니라 **나한테 유용한 부분만** 추출
- 저자가 제시하는 사고 모델, 분석 틀, 의사결정 체계
- 최대 3개 프레임워크 추출

### 적용 섹션
- 각 프레임워크로 내 현재 상황(프로젝트, 커리어, 투자 등)을 분석
- "이 프레임워크에 따르면 나는 ___해야 한다"

### 글감 가능성
- 독후감 자체가 X/블로그 글감이 될 수 있는지 평가
- 글감이 될 경우 핵심 메시지와 구조 제안

## 저장

```yaml
---
type: knowledge
source_type: book
source_quality: primary
confidence: high
verification_status: verified
counter_scenario: "프레임워크 한계"
applicability: "내 상황 적용 요약"
verified_date: YYYY-MM-DD
tags: []
related_topics: []
book_meta:
  title: "책 제목"
  author: "저자"
  key_frameworks: ["프레임워크1", "프레임워크2"]
---

# 핵심 프레임워크

## 프레임워크 1: {이름}
{설명}

### 내 상황에 적용하면?
{적용 분석}

# 검증 결과

## [사실]로 확인된 주장
- ...

## [해석]인 부분
- ...

# 인상적인 문장
> "원문 인용"
```

파일명: `{저자}_{책제목요약}_{YYYYMMDD}.md`
저장 위치: `knowledge/{도메인}/`
