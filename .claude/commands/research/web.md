# 웹 리서치 + 검증

웹 검색 기반 리서치를 수행하되, AI 웹 검색의 한계(할루시네이션)를 인식하고 검증 로직을 적용한다.

## 핵심 원칙

> **1개 소스만으로 사실 판단 금지. 복수 소스 교차 검증 필수.**

## 검증 로직

### 소스별 분류
모든 검색 결과에 대해:
- `[사실]` — 복수 신뢰 소스에서 확인됨
- `[해석]` — AI 추론 또는 단일 소스 분석
- `[불명]` — 확인 불가, 출처 없음

### 확신도 표기
- 🟢 **높음** — 복수 신뢰 소스 일치 (공식 기관, 학술자료 등)
- 🟡 **중간** — 단일 신뢰 소스 또는 복수 비공식 소스
- 🔴 **낮음** — 추정, AI 생성 가능성, 확인 불가

## 리서치 프로세스

### 1. 검색 실행
- 핵심 키워드로 웹 검색
- 최소 3개 이상 소스 확인 시도

### 2. 교차 검증
- 동일 사실을 다루는 복수 소스 비교
- 소스 간 불일치 시 명시적으로 표기

### 3. 결과 구조화
```
## 리서치 결과: {주제}

### 확인된 사실 (🟢)
- [사실] {내용} — 소스: {소스1}, {소스2}

### 단일 소스 정보 (🟡)
- [해석] {내용} — 소스: {소스}

### 미확인 정보 (🔴)
- [불명] {내용} — 확인 불가 사유: {사유}

### 소스 목록
1. {URL} — 신뢰도: {높음/중간/낮음}
2. ...
```

### 4. 저장 여부 결정
- **자동 저장하지 않음** — 사용자에게 결과를 보여주고 저장 여부를 물어봄
- 사용자가 저장을 원하면:
  ```yaml
  ---
  type: knowledge
  source_type: web_research
  source_quality: secondary
  confidence: high | medium | low  # 종합 확신도
  verification_status: verified | partial
  verified_date: YYYY-MM-DD
  tags: []
  related_topics: []
  ---
  ```
- 파일명: `research_{주제요약}_{YYYYMMDD}.md`
- 저장 위치: `knowledge/{도메인}/`

## 주의사항
- "확실합니다" 같은 표현 금지 — 항상 확신도와 함께 전달
- AI가 생성했을 가능성이 있는 정보는 반드시 `[불명]` 처리
- 시간에 민감한 정보(환율, 주가 등)는 검색 시점 명시
