# 4단계 비판적 검증 (단독 실행)

기존 노트나 특정 주장에 대해 4단계 검증을 독립적으로 실행한다.

## 사용 시나리오
- "이 주장 검증해줘"
- "이 노트 다시 검증해볼까"
- "이 정보 신뢰할 수 있어?"
- 기존 `verification_status: unverified` 노트 재검증

## 검증 프로세스

### ① 출처 검증
> 이 정보는 어디서 왔는가?

- **1차 자료**: 원저자, 공식 기관, 학술 논문, 공식 데이터
- **2차 자료**: 뉴스 기사, 분석 리포트, 블로그 해석
- **3차 자료**: AI 요약, SNS 전파, 출처 불명
- → `source_quality` 판정

### ② 근거 검증
> 이 주장을 뒷받침하는 근거는 무엇인가?

각 핵심 주장에 대해:
- `[사실]` — 복수 소스에서 확인된 데이터/사실
- `[해석]` — 분석가/저자의 추론이나 해석
- `[불명]` — 출처나 근거를 확인할 수 없음

### ③ 반대 시나리오
> 이 주장이 틀릴 수 있는 조건은?

- 어떤 전제가 바뀌면 결론이 달라지는가?
- 반대 논거를 제시하는 신뢰할 수 있는 소스가 있는가?
- → `counter_scenario` 필드에 기록

### ④ 적용성
> 내 상황에 해당되는가?

- 이 정보가 나의 의사결정에 영향을 주는가?
- 시간적 유효성 (이 정보는 언제까지 유효한가?)
- → `applicability` 필드에 기록

## 종합 판정

검증 완료 후 종합 판정:

| 판정 | 기준 | confidence |
|------|------|-----------|
| 🟢 높음 | 복수 1차 소스 일치, 데이터 기반 | high |
| 🟡 중간 | 단일 신뢰 소스, 또는 일부 해석 포함 | medium |
| 🔴 낮음 | 추정/의견, 출처 불명, 반대 논거 강력 | low |

## 결과 반영

검증 대상이 기존 노트인 경우 프론트매터 업데이트:
```yaml
verification_status: verified  # 또는 partial
confidence: high | medium | low
counter_scenario: "업데이트된 반대 시나리오"
applicability: "업데이트된 적용성"
verified_date: YYYY-MM-DD
```

검증 대상이 새로운 주장인 경우 결과를 사용자에게 보여주고, 저장 여부는 사용자가 결정.
