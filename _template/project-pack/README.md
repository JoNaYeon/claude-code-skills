# Project Pack Templates Staging

> 이 폴더는 requirement, PRD, prototype review 같은 PM 산출물 템플릿을
> 실제 Vault 반영 전에 먼저 버전 관리하는 staging 공간이다.

목표 성격:
- 작업 단위 planning artifact 템플릿
- `CONTEXT.md` / `PROGRESS.md`를 대체하지 않는 별도 계층
- Vault project pack / Vault tasks 정본을 위한 contract staging

예상 장기 홈:
- `/Users/gayoungji/Dev/brain-vault/_templates/project-pack/`

현재 포함된 템플릿:
- `00_Project Context.md`
- `01_Working Agreement.md`
- `02_Requirement Intake.md`
- `03_PRD.md`
- `03-1_Feature Brief.md`
- `04_Scope Options.md`
- `05_Prototype Brief.md`
- `06_Prototype Review.md`
- `07_Decision Log.md`
- `08_Task Item.md`
- `09_Batch Run.md`

운영 규칙:
- PM 산출물의 frontmatter나 section contract가 바뀌면 이 staging본부터 수정한다
- Project Pack 원문은 허브 문서가 아니라 작업 문서다
- live source of truth는 저장소 안 sample pack이 아니라 실제 Vault project pack / Vault tasks다
- 장기 문서 승격은 별도 promotion rule에 따라 요약만 반영한다
