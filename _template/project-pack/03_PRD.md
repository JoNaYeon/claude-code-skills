# PRD Summary

> UX 중심 기획 산출물은 [[03-1_Feature Brief]] 를 사용한다. PRD는 기능 전체의 요구사항/수락 기준을 정의할 때 사용한다.

## Acceptance Criteria

각 요구사항의 수락 기준을 Given-When-Then 형식으로 기술합니다.

### R1: 팀별 휴가 현황
- [ ] **Given** HR 관리자가 대시보드에 접속했을 때, **When** 페이지가 로드되면, **Then** 전체 팀의 이번 달 휴가가 캘린더에 표시된다.
- [ ] **Given** 휴가가 없는 날이 있을 때, **When** 해당 날짜를 보면, **Then** 빈 셀이 표시된다 (에러 아님).

### R2: 필터링
- [ ] **Given** 부서 필터에서 "개발팀"을 선택했을 때, **When** 필터가 적용되면, **Then** 개발팀 소속 직원만 표시된다.
