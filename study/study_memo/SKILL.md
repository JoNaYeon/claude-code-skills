# 📝 Study — Memo to Notion

사용자가 기억해야 할 메모(파일·링크·텍스트·영상 등)를 분석하여 **Notion MEMO DB**에 속성 맞춤 페이지로 정리합니다.

## 대상 Notion DB

- **DB 이름**: MEMO
- **URL**: https://www.notion.so/3450a1289f1680fdb4f7cc12780ea0c3
- **Data Source ID**: `3450a128-9f16-80a0-b066-000b2e231912`

### 스키마

| 속성 | 타입 | 설명 |
|------|------|------|
| `이름` | title | 페이지 제목 (필수) |
| `속성` | multi_select | idea / news / book / note / work / ect |
| `상위 항목` | relation | 같은 DB 내 부모 페이지 (옵션) |
| `하위 항목` | relation | 같은 DB 내 자식 페이지 (옵션) |
| `생성 일시` | auto | 자동 |
| `최종 편집 일시` | auto | 자동 |

## 실행 순서

### 1. 📥 입력 수집

사용자가 제공하는 것을 받습니다 — 아래 중 하나 이상:
- **URL** (기사, 블로그, 영상, GitHub, 트윗 등)
- **파일** (PDF, 이미지, 문서)
- **텍스트** (발췌 문단, 메모, 링크 없는 글)
- **주제명** (예: "OAuth 2.0 메모해줘")

### 2. 🏷 속성(카테고리) 판단

내용의 성격을 분석하여 **`속성`** 필드 값을 결정. 우선순위 높은 것부터:

| 속성 | 언제 사용 | 예시 |
|------|-----------|------|
| **idea** 💡 | 내가 떠올린 아이디어, 구현 가능성 있는 씨앗 | "앱에 이런 기능 추가하면 좋겠다" |
| **news** 📰 | 기사, 뉴스, 산업 동향, 릴리즈 공지 | Anthropic 신모델 발표, 기사 링크 |
| **book** 📚 | 책 발췌, 독서 메모, 서평 | "『팩트풀니스』 4장 요약" |
| **note** 📝 | 일상에서 쓰는 리스트/체크사항/개인 메모 | 장보기 리스트, 준비물, 체크리스트, 여행 짐 |
| **work** 💼 | 업무 관련 할 일, 회의 메모, 업무 정보 | "김대표 미팅에서 나온 3가지" |
| **ect** 🗂 | 위에 해당하지 않는 기타 | 레시피, 쇼핑 리스트 등 |

→ 애매하면 **사용자에게 1초 확인** 후 진행.

### 3. 🔬 내용 수집 & 출처 확보

- URL이면: `WebFetch`로 원문 로드
- 파일이면: `Read`로 로드
- 필요시 `WebSearch`로 **참고 사이트 추가 조사** (관련 공식 문서, 비교 자료 등)
- 모든 출처는 **마지막 섹션에 링크**로 기록

### 4. ✨ 속성별 맞춤 템플릿으로 정리

속성에 따라 섹션 구성을 다르게 합니다.

#### 💡 idea
```
## 🎯 문제 인식 / 동기
## 💡 핵심 아이디어 (1~2문장)
## 🛠 실현 방향
## ⚠️ 제약·리스크
## ✅ 다음 액션 (TODO)
## 🔗 참고
```

#### 📰 news
```
## 📌 한 줄 요약
## 🔎 핵심 내용 (bullet 3~5개)
## 🌐 맥락 / 왜 중요한가
## 💬 내 시각 / 시사점
## 🔗 원문 링크
## 📚 함께 보기 (관련 기사/문서)
```

#### 📚 book
```
## 📖 서지 정보 (제목 / 저자 / 출간 / 장-절)
## 🎯 핵심 주장
## ✍️ 인상 깊은 구절 (인용 + 페이지)
## 💭 내 생각 / 적용할 점
## 🔗 관련 자료
```

#### 📝 note (일상 메모 / 리스트)
```
## 📋 체크리스트 (있으면)
- [ ] 항목 1
- [ ] 항목 2

## 📝 메모 / 상세 (필요 시)
## 🗓 일정 / 장소 (있을 경우)
## 🔗 참고 링크 (있으면)
```

> 장보기·쇼핑 리스트·준비물·할 일 등 실용 메모 위주로 **짧고 훑기 좋게**. 학습/개념 정리는 이 속성이 아님 (학습은 `/study_engineering-note` 또는 `ect`).

#### 💼 work
```
## 🗓 맥락 (언제 / 어디서 / 누구와)
## 📌 핵심 내용 (요점 bullet)
## ✅ 할 일 (담당자 / 데드라인)
## ⚠️ 의사결정 필요 사항
## 🔗 관련 문서/링크
```

#### 🗂 ect
```
## 📌 내용
## 🔗 참고 (있으면)
```

> **규칙**: 해당 속성 템플릿이 기본. 내용에 따라 섹션 추가/삭제 자유롭게.

### 5. 🔗 하위 페이지 or 상위 항목 연결 여부 판단

- 기존 메모와 연관 있는 주제면 `notion-search`로 먼저 검색
- 유사 페이지 있으면 **상위 항목** 또는 **하위 항목**으로 연결 제안 (승인 후 진행)
- 주제가 크면 하위 페이지로 분할 제안 (`study_engineering-note`와 동일한 승인 룰)

### 6. 📤 Notion 페이지 생성

`notion-create-pages` 호출:
- **parent**: `data_source_id` = `3450a128-9f16-80a0-b066-000b2e231912`
- **properties**:
  - `이름`: 명확하고 검색하기 좋은 제목
  - `속성`: `["idea"]` 등 JSON array 형식
- **content**: 위 템플릿 기반 본문

### 7. ✅ 결과 보고

- 생성된 페이지 URL
- 적용한 속성
- 포함 섹션 요약
- 추가 질문: "다른 메모 더 있나요?"

## 제목 작성 원칙

제목은 **나중에 검색할 때 기억할 키워드**를 포함:
- ❌ `"글 정리"`, `"생각"`
- ✅ `"OAuth 2.0 Refresh Token 순환 전략"`
- ✅ `"[책] 팩트풀니스 4장 — 두려움 본능"`
- ✅ `"[뉴스] Anthropic Claude 4.7 1M context 공개 (2026-04)"`

권장 포맷: `[대괄호 태그] 핵심 주제 — 부제`

## 작성 원칙

1. **사용자가 **3개월 뒤 다시 봐도** 이해 가능할 것** — 맥락 보존
2. **출처 필수** — URL이 있었다면 반드시 링크 (news/book/work에서 특히)
3. **과도한 LLM 재해석 금지** — 원문 인용 + 내 생각 분리
4. **중복 방지** — 생성 전 `notion-search`로 같은 제목 확인
5. **민감 정보 차단** — 비밀번호, API 키, 개인정보 포함 금지

## 주의사항

- `속성` 값은 반드시 JSON array 문자열: `["idea"]`, `["news", "work"]` (복수 선택 가능)
- 제공된 자료 외에 **추측 기반 보충 금지** — 필요하면 WebSearch로 검증된 추가 자료만
- 페이지 **길이는 1분 내 훑을 수 있게** — 장문이면 하위 페이지로 분할 제안
